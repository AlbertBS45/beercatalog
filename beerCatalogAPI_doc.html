Beer Catalog API

General notes

This API manages a beer catalogue from different manufacturers.

Beer consumers can look up the beer catalogue in order to inspire future purchases.

API detail

Beers

The operations available are:

  • Obtain information by any beer attribute
  • Add a new beer
  • Update an existing beer
  • Delete a beer

Get all beers

Get all beers from the catalog.

By default, page size is 10 elements and results are sorted by id in ascending order.

This request can be performed without any authentication.

  • HTTP

Create a beer

Creates a new beer from a well-formed JSON with the required fields.

Required fields are:

  • name
  • description
  • abv
  • manufacturer_id

The attributes manufacturer_name and manufacturer_nationality will be ignored if specified.

Admin user can create a new beer from any manufacturer.

Manufacturer user can create a new beer only if it is its own manufacturer.

Anonymous user can not create a new beer.

If using Postman when changing between users, do not forget to clean cookies between each user test case.

  • Curl
  • HTTP
curl -X POST -d '{
    "name": "Equilater",
    "type": "Stout",
    "description": "description text...",
    "abv": 5.0,
    "manufacturer": {
        "id": 2
    }
}' "http://localhost:8080/beercatalog/beers"

  • id
  • name
  • description
  • abv

Admin user can update a beer from any manufacturer.

Manufacturer user can update a beer only if it is its own manufacturer.

Anonymous user can not update a beer.

If using Postman when changing between users, do not forget to clean cookies between each user test case.

Create a new manufacturer

Creates a new manufacturer from a well-formed JSON with the required fields.

Required fields are:

  • name
  • nationality

Admin user can create a new manufacturer. This action will trigger a user creation related to the new manufacturer. The default credentials for the new user follow this pattern:

  • username: {newManufacturerName}@provider.com
  • pwd: provider

This new user will also be assigned with the role “ROLE_MANUFACTURER”.

Eg: Manufacturer created with name “damm” will create a user with the following credentials:

  • damm@provider.com
  • provider

Manufacturer can not create a new manufacturer.

Anonymous user can not create a new manufacturer.

If using Postman when changing between users, do not forget to clean cookies between each user test case.

  • Curl
  • HTTP
curl -X POST -d '{
    "name": "Damm",
    "nationality": "Spain"
}' "http://localhost:8080/beercatalog/manufacturers"
